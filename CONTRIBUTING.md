# Contributing to ifonly.muse

`ifonly.muse` 는 if-only Echo 서비스의 *개인 사용자용 PC 동반 도구* 로 운영됩니다. 외부 기여 (PR · 이슈 · 기능 제안) 는 현재 정기적으로 응대하지 않습니다.

이 입장은 적대적 배제가 아니라 운영 현실입니다 — Echo 본 서비스의 빠른 변화에 muse-agent 가 따라붙는 형태로 개발되며, 외부 기여를 안전하게 통합·테스트할 인프라가 갖춰져 있지 않습니다.

---

## 보안 취약점 제보

코드를 읽다가 보안 취약점 (인증 우회, 자격증명 노출, 임의 파일 접근 등) 을 발견하셨다면, **공개 이슈 대신** 다음 채널로 제보를 부탁드립니다:

- **이메일**: 본 repo 의 commit author 주소 (`git log --format='%ae' -n1` 로 확인 가능) 또는 GitHub profile 의 공개 연락처
- 제목 머리: `[ifonly.muse security]`
- 가능하면 재현 절차 · 영향 범위 · 임시 완화책 포함

응답은 보통 7 영업일 이내. 수정 후 본 repo 에 fix 가 push 되면 제보자에게 알려드립니다 (원하시면 commit 메시지에 credit 표기).

---

## 일반 이슈 / 기능 제안

- GitHub Issue 는 열어두지만, 즉각적인 응답은 보장하지 않습니다.
- 사용 중 발견한 버그 · 오작동 보고는 환영합니다 (재현 환경 · 로그 발췌 포함 시 우선 처리).
- 큰 기능 제안은 *검토 후 답신 없을 수 있음* 을 양해 부탁드립니다.

---

## Pull Request

본 repo 는 외부 PR 을 정기적으로 머지하지 않습니다. 다음 경우는 예외적으로 검토합니다:

1. 위에서 제보한 보안 취약점에 대한 fix
2. README · CHANGELOG · 코드 주석의 명백한 오타 / 사실 오류
3. 명시적으로 "PR 환영" 이라고 응답한 이슈에 대한 구현

PR 을 보낼 경우:

- 한 commit = 한 가지 변경 (logical change)
- 메시지는 `area: 변경 요지` 형식 (e.g. `docs: fix typo in README`)
- 가능하면 영어 또는 한국어 메시지 (혼용 OK)
